1. Tool Kya Hai? (@langchain/core/tools)
Socho AI ek dimaag hai, par uske paas haath nahi hain. Tool wo "haath" (Function) hai jo tum AI ko dete ho.

Logic: Jab tum tool() function use karte ho, toh tum sirf ek JavaScript function nahi likh rahe, tum AI ko us function ka Description aur Instructions bhi de rahe ho.

AI Kaise Samjhta Hai? AI tumhara code nahi padhta. Wo description padhta hai. Agar user bolega "Mujhe iPhone dikhao", toh AI dekhega: "Hmm, mere paas ek tool hai jiska description 'Search for products' hai, mujhe ise call karna chahiye."

2. Zod Kya Hai? (zod)
Zod ek Schema Validation library hai. Iska kaam hai "Chaukidaari" karna.

Kyun Chahiye? AI kabhi-kabhi behki-behki baatein karta hai. Maan lo qty (quantity) mein usne number ki jagah "five" (string) bhej diya. Agar wo tumhare API pe gaya, toh API crash ho jayega.

Zod Ka Role: z.object({ productId: z.string() }) ye line AI ko majboor karti hai ki:

productId bhejna hi bhejna hai (Required).

Wo sirf string honi chahiye.

Working: Agar AI galat format mein data bhejta hai, toh LangChain wahi rok dega aur AI ko bolega: "Bhai, sahi format mein data bhej, ye galat hai."


1. Tool ka Execution Flow (Step-by-Step)
AI ko tumhare function ke andar ka code (axios.get) nazar nahi aata. Uske liye flow aisa hota hai:

AI Reading Phase: Sabse pehle AI Description padhta hai. Wo dekhta hai ki use "Shoes" dhoondne hain, toh searchProduct ka description match kar jata hai.

Input Generation: AI dimaag lagata hai aur ek object banata hai: { "query": "blue shoes" }.

Zod Validation (Chaukidaar): Ab Zod active hota hai. Wo check karta hai ki AI ne jo object bheja hai, kya wo tumhare schema (z.object) se match karta hai?

Agar AI ne query ki jagah search_term likh diya, toh Zod use error de dega aur AI ko bolega "Sahi format mein bhej bhai!"

Async Logic Execution: Agar Zod ne "Green Signal" de diya, tab tumhara async wala code (Axios call) chalta hai.

Data Return to AI: API se jo data aaya, wo String ban kar AI ke paas jata hai.

AI Decision: AI us string ko padhta hai.

Case A (Success): AI ko product mil gaya, wo user ko reply de dega "Bhai, ye rahe tumhare shoes".

Case B (Fail/Retry): Agar API ne khali array [] bheja, toh AI dekhega "Result toh khali hai". Phir wo wapas loop karega aur shayad kisi aur tool se ya thoda alag query se dhoondhne ki koshish karega (Yahi LangGraph ki taqat hai).

2. z.object ka kya matlab hai?
Simple shabdon mein: z.object ka matlab hai ek "Structure" ya "Sancha" (Template).

Jaise ek form hota hai jahan fixed jagah hoti hai Name, Age aur Email bharne ki, waise hi z.object batata hai ki AI jo data bhejega wo ek Object {} hona chahiye.

1. Tool ke baad ka Step-by-Step Flow
Tools Node ka Kaam Khatam: Tool ne API call ki, data laya aur use ToolMessage bana kar state mein daal diya.

Edge Trigger: Tumne likha hai .addEdge("tools", "chat"). Iska matlab hai ki jaise hi tool ka kaam khatam ho, control wapas chat node ko de do.

Chat Node (Round 2): Ab chat node phir se shuru se execute hoga. Lekin is baar uske paas Extra Information hai (jo tool ne abhi-abhi la kar di hai).

Re-Checking Logic: chat node dubara model.invoke karega. Model saari history dekhega:

User: "Sasta iPhone dhoondo."

AI (Pehle wala): "Mujhe tool chahiye."

Tool: "Ye raha sasta iPhone ID: 123, Price: 50,000."

Faisla (The Decision): Model ab dekhega ki uske paas data aa chuka hai. Wo sochega: "Ab mujhe kisi aur tool ki zaroorat hai? Nahi! Mere paas info hai."

Response Generation: Is baar AI tool_calls nahi bhejega, balki ek seedha text message bhejega: "Bhai, mujhe 50,000 mein iPhone mil gaya hai!"


Step-by-Step Logic:

Registry: Jab tum model.invoke karte ho, LangChain tumhare saare tools ka name, description, aur schema uthata hai aur use ek format (JSON Schema) mein badal kar Google Gemini (AI) ko bhej deta hai.

AI Choice: AI dekhta hai—"Accha, mere paas searchProduct naam ka ek tool hai jo product search karta hai."

Response: Jab AI ko wo tool chalana hota hai, toh wo wahi same name wapas bhejta hai jo tumne use diya tha.

Your Code (Matching): Phir tumhare tools node mein tum ye line chala rahe ho:
const tool = tools[call.name]
Yahan call.name wahi "searchProduct" hai jo AI ne bheja, aur tools tumhara wo object hai jisme tumne tools export kiye hain. Dono match kar jate hain aur sahi function execute ho jata hai.

const {tool} = require('@langchain/core/tools')
const {z} = require("zod")
const axios = require('axios')

//product serach and cart mei add krna hai 
                                        //token ai ko bej ke generate ya change nahi krwaana toh metadata ke through bejege 
const searchProduct = tool(async({query,token})=>{

    const response  = await axios.get(`http://localhost:3001/api/products?q=${query}`,{
        headers:{
            Authorization: `Bearer ${token}`
        }
    })

    return  JSON.stringify(response.data);
    //final me saare hi  string mei hi return ho kyu ki AI string hi read krta hai aur AI ko object smj nahi aate
},{
    name:"searchProduct",
    description: "Search for products based on a query",
    schema: z.object({
        query: z.string().describe("The  search query for products")
    })
})


const addProductToCart = tool(async ({productId, qty=1, token})=>{

    const response = await axios.post(`http://localhost:3002/api/cart/items`,{
        productId,
        qty 
    },{
        headers:{
            Authorization: `Bearer ${token}`
        }
    })

    return `Added product with id ${productId} (qty:${qty}) to cart  `

},{
    name:"addProductToCart",
    description: "Add a  product to the shopping cart",
    schema:z.object({
        productId: z.string().describe("The id of the product to add to the cart "),
        qty:z.number().describe("The quantity of the product to add to the cart").default(1),
    })
})


module.exports = {
    searchProduct,addProductToCart
}
