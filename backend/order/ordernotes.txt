Jab aap roles = ["user"] likhte ho, toh ye sirf tab apply hota hai agar developer ne function call karte waqt koi argument nahi bheja.
Hacker ke hath mein control nahi hai: Ye code server-side par chalta hai. Hacker browser se ye function call nahi kar sakta. Ye aap (the developer) decide karte ho ki router mein kaunsa middleware lagana hai.

Safety Net: Agar aap galti se kisi route par role pass karna bhool jayein, toh system usse bypass nahi karne dega. Wo usse "user" (sabse basic level) maan lega, na ki "admin".

async function getOrderById(req, res) {
    const user = req.user;
    const orderId = req.params.id

    try {
        const order = await orderModel.findById(orderId);
        const isOwner = order.user.toString() === user.id;
        const isAdmin = user.role === 'admin';
        if (!order) {
            return res.status(404).json({
                message: "Order not found"
            })
        }

        if (order.user.toString() !== user.id) {
            return res.status(403).json({
                message: "Forbidden: You don't have access"
            })
        }

        res.status(200).json({
            order
        })

    } catch (err) {
        res.status(500).json({
            message: "Internal server error", error: err.message
        })
    }
}





-> orderModel.create() vs new orderModel()

Toh Mongoose usi waqt database mein data save kar deta hai. create() method do kaam ek saath karta hai: ek naya instance banana aur use database mein save() kar dena. Isliye aapko alag se save karne ki zaroorat nahi padi.
const order = await orderModel.create({ ... });

const order = new orderModel({ ... }); // Sirf memory mein object bana
// Yahan order.save() zaroori hota database mein bhejne ke liye

order.shippingAddress.city = 'Jaipur';
await order.save(); // Changes save karne ke liye
